Student Management System – Intermediate State
This program is at the Intermediate Level, as the name suggests.
It is built on top of the earlier Beginner State version, but now comes with richer features and more flexibility.
It still doesn’t have truly powerful algorithms or advanced input handling yet — those will come in later versions.
Think of this as the next step forward in the evolution of the project.

 What This Program Contains

All baseline features from the Beginner version
Updating and adding student functions
Sorting algorithms (by ID, Name, or Marks)
Displaying total number of students
Auto‑calculation of average marks (dynamic updates after add/update/delete)
Multi‑Subject Mark System (foundation laid for future expansion)

🚫 What This Program Does Not Contain

Powerful algorithms
Powerful database
Strong admin authentication system
…and other advanced features (coming soon in later levels)

🧭 Step‑by‑Step Mind Map (Notes)

"Student Management System - Intermediate" : {
    "Option A" : "Search Student by Name or ID",
    "Option B" : "Calculate Average Marks (auto, dynamic)",
    "Option C" : "Find Highest and Lowest Marks",
    "Option D" : "Grade System (A/B/C/F)",
    "Option E" : "Edit Student Data (Delete/Update/Add)",
    "Option F" : "Student Information (Sort by ID, Name, Marks & Display Total Students)",
    "Option G" : "Multi-Subject Mark System",
    "Option Out" : "Exit the Program"
}

Option A – Search Student

Functions to search students by name or ID.
Displays student details including marks.
Reusable search functions improve maintainability and avoid duplication.

Option B – Average Marks

Average marks are automatically recalculated whenever the student list changes.
Keeps results dynamic and always up to date.

Option C – Highest / Lowest Marks

Identifies and displays the student with the highest and lowest marks.
Simple but effective comparison logic.

Option D – Grade System

Assigns grades (A/B/C/F) based on marks.
Displays pass/fail status for each student.

Option E – Edit Student Data

Delete student by ID.
Update student’s name or mark.
Add new student (with duplicate ID prevention).
Expands the system into a more practical management tool.

Option F – Student Information

Display all students with details.
Sort students by ID, Name, or Mark using std::sort.
Display total number of students.

Option G – Multi‑Subject Mark System

Each student can store marks for multiple subjects (e.g., Math, English, Science).
Functions calculate per‑subject averages and overall averages.
Sorting and grading can be extended to multi‑subject context.

Option Out – Exit
Cleanly exits the program when the user chooses.

📝 Notes
This intermediate version focuses on improving usability and adding richer features.
It introduces sorting, editing, and dynamic averages, while laying the groundwork for multi‑subject marks.
Future versions will include:

File handling (save/load student data for persistence).

More advanced algorithms.

Stronger input validation and error handling.

Admin authentication system.
